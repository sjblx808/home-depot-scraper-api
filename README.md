```python
import requests

API_KEY = "你的key"
target_url = "https://www.homedepot.com/p/产品路径/SKU编号"

    f"http://api.scraperapi.com?api_key={API_KEY}&url={target_url}&render=true"
```

## 技术细节：让采集更稳的几个参数

ScraperAPI针对电商网站有几个关键参数值得了解：

**`render=true`**：启用无头浏览器渲染。Home Depot的价格、库存、促销标签都依赖JS加载，不开这个参数你拿到的是空壳HTML。开启后每次请求消耗10个API credits（普通请求消耗1个），但对Home Depot来说这是必须的。

**`country_code=us`**：指定代理出口国家。抓Home Depot必须用美国IP，否则会被重定向或直接403。

**`session_number`**：保持会话一致性。如果你需要模拟登录态或者连续翻页，用同一个session number可以让请求走同一个代理IP，避免被检测为异常流量。

**`autoparse=true`**：ScraperAPI对部分电商站点提供结构化JSON输出。虽然Home Depot不在官方autoparse支持列表的第一梯队，但配合render参数，返回的HTML质量足够用XPath或CSS选择器稳定提取。

## 常见疑问

### Q1：ScraperAPI抓Home Depot会被封号吗？

不会封你的账号——因为你根本不需要登录Home Depot。ScraperAPI用的是它自己的代理IP池，被封的是代理IP，它会自动轮换，这个过程对你完全透明。我跑了18个月，从没收到过Home Depot的任何警告。

### Q2：失败的请求会扣费吗？

不会。ScraperAPI只对成功返回200状态码的请求计费。如果请求失败（超时、被拦截、服务器错误），不消耗你的API credits。它内部会自动重试最多3次，全部失败才返回错误码给你。

### Q3：能抓Home Depot的用户评论吗？

能。评论区是动态加载的，需要开启`render=true`并且等待页面完全渲染。我的做法是加一个`wait_for_selector`参数（或者在请求后等待3~5秒），确保评论区DOM加载完毕再解析。

### Q4：每月5000次免费额度够测试吗？

够验证技术可行性。5000次请求如果都开render（每次消耗10 credits），实际能抓500个页面。用来跑通流程、验证解析逻辑、确认数据质量完全足够。确认可行后再升级付费套餐。

### Q5：支持哪些编程语言？

ScraperAPI本质是HTTP API，任何能发HTTP请求的语言都能用——Python、Node.js、Java、Go、PHP、Ruby，甚至cURL直接命令行调用。官方提供Python和Node.js的SDK封装，但直接用requests库调用也就一行代码的事。

### Q6：抓取速度怎么样？

取决于你的套餐并发数和是否开启渲染。不开渲染的普通请求平均响应时间2~5秒，开渲染的请求8~15秒（因为要等JS执行完）。10并发跑Home Depot详情页，每小时能稳定产出2000~3000个页面数据。

## 什么情况下ScraperAPI不适合你

说几个不适合的场景，免得你花冤枉钱：

- **实时性要求极高**（秒级响应）：ScraperAPI的渲染请求需要8~15秒，如果你的业务需要毫秒级价格更新，它不合适，你需要直接对接Home Depot的官方API（如果有权限的话）。
- **日采集量超过千万级**：这个量级自建方案的边际成本更低，ScraperAPI更适合百万级以下的场景。
- **只需要抓一次性数据**：如果只是一次性导出几百个商品信息，手动复制或者用浏览器插件可能更快。
